# Design Integrity Guard

## 觸發條件
任何涉及用戶/玩家可見文字的任務：文案撰寫、UI 文字、系統訊息、錯誤提示、引導文字、故事內容、通知文案、商店文案。

---

## 啟動步驟

1. 讀取 `docs/design-principles.md`（若未在 context 中）
2. 確認當前任務的內容類型
3. 載入對應的判準區塊

---

## 共通判準（所有內容類型適用）

### 語氣底色
從 `design-principles.md` 的「用戶應該感受到什麼」提取。

一句話判準：**讀你的 design-principles.md「最終判準」欄位，每段文字都必須通過。**

### 禁用詞彙
從 `design-principles.md` 的「禁用詞彙」提取。

### 反模式
從 `design-principles.md` 的「絕對不能變成什麼」提取。任何產出如果讓產品更接近這些反模式，退回重寫。

## 各內容類型判準

以下為模板。請根據你的產品替換範例內容。

### 引導/開場文字
- [ ] 不做功能導覽、不解釋系統
- [ ] 讓用戶先體驗、先感受，再理解
- [ ] 語氣像產品在等你，不是教學在教你
- 自訂：___

### 角色/物件介紹
- [ ] 不寫功能標籤、不寫能力數值
- [ ] 寫的是這個角色/物件的「氣質」
- [ ] 讀起來讓人想認識，不是想比較
- 自訂：___

### 對話/互動文字
- [ ] 關係深化的方向是___（例：越安靜越有重量 / 越熟越隨性）
- [ ] 禁止的方向是___（例：越來越撒嬌 / 越來越像客服）
- 自訂：___

### UI 文字
- [ ] 按鈕用動作而非狀態（「前往」而非「確認前往」）
- [ ] 系統提示用產品世界的語言，不用系統語言
- [ ] 不用驚嘆號
- 自訂：___

### 通知/推播文字
- [ ] 變化要「被發現」而不是「被公告」
- [ ] 不刺激、不催促
- 自訂：___

### 商店/付費文字
- [ ] 不推銷、不製造焦慮
- [ ] 付費體驗讓用戶感到___，而非「在買東西」
- 自訂：___

## 交付前自檢

每段用戶可見文字交付前逐條檢查：

1. 有沒有用到禁用詞彙？
2. 讀起來像系統通知還是像產品世界在說話？
3. 有沒有不自覺地寫成促銷/遊戲化/客服語氣？
4. 有沒有違反 design-principles.md 的任何生死線？
5. **最終判準：如果這段文字被截圖傳出去，會不會讓人覺得「這就是又一個 ___（你最不想被歸類的東西）」？**

第 5 條過不了就退回重寫。

---

## 守衛模式（審查其他 agent 產出）

被 PM 派工審查時：

1. 載入此 skill
2. 逐段檢查產出物中的用戶可見文字
3. 比對禁用詞彙
4. 比對反模式
5. 通過 → 回報 PM
6. 不通過 → 標註具體問題 + 引用違反的判準條目 → 退回修改

標註格式：
```
🚫 [文字段落] — 違反：[判準條目] — 建議方向：[一句話]
```
